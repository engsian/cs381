CSS Syntax

The building blocks of styling

CSS Rule Structure

/* Selector targets HTML elements */
selector {
    property: value;    /* Declaration */
    property: value;
}

/* Example */
h1 {
    color: #2563eb;
    font-size: 2rem;
    margin-bottom: 1em;
}
Selector — What to style
Property — What aspect
Value — How to style it

Three Ways to Add CSS

1. External (Recommended) 
<link rel="stylesheet" href="styles.css">
Separate file, reusable, cacheable
2. Internal 
<style> h1 { color: blue; } </style>
In <head>, page-specific
3. Inline (Avoid) 
<h1 style="color: blue;">Title</h1>
Hard to maintain, highest specificity

CSS Selectors

element — All of type 
p { color: gray; }
.class — By class name 
.btn { padding: 10px; }
#id — By ID (unique)

parent child — Descendant

:pseudo-class — State 
a:hover { color: red; }
[attr] — By attribute 
[type="text"] { border: 1px; }

box-sizing: border-box

The most important CSS rule you'll write

content-box (default)

width = content only

width: 200px + padding + border = larger!

border-box (recommended)

width = content + padding + border

width: 200px = exactly 200px

/* Apply to all elements - put this at the top of your CSS */
*, *::before, *::after {
    box-sizing: border-box;
}

CSS Units

Unit Type Use Case
px Absolute Borders, small fixed sizes
% Relative to parent Widths, responsive layouts
em Relative to parent font Spacing relative to text
rem Relative to root font Font sizes, consistent spacing
vw/vh Viewport % Full-screen sections

Tip: Use rem for fonts, px for borders, % or fr for layouts

Flexbox vs Grid

Flexbox

  • One direction at a time
  • Content-driven sizing
  • Great for: navbars, cards, centering

Grid

  • Rows AND columns
  • Layout-driven sizing
  • Great for: page layouts, galleries

Rule of thumb: Grid for page layout, Flexbox for components

Responsive Design

One website, every screen size

Media Queries

/* Base styles (mobile first) */
.container { padding: 1rem; }

/* Tablet and up */
@media (min-width: 768px) {
    .container { padding: 2rem; }
}

/* Desktop and up */
@media (min-width: 1024px) {
    .container {
        max-width: 1200px;
        margin: 0 auto;
    }
}

Mobile-First Approach

Desktop-First

/* Desktop styles */
.nav { display: flex; }

/* Override for mobile */
@media (max-width: 768px) {
    .nav { display: block; }
}

Mobile-First

/* Mobile styles (default) */
.nav { display: block; }

/* Enhance for desktop */
@media (min-width: 768px) {
    .nav { display: flex; }
}

BEM Methodology

Organizing CSS at scale

BEM: Block-Element-Modifier

A naming convention for maintainable CSS

Block — Standalone component: .card

Element — Part of block: .card__title

Modifier — Variation: .card--featured

/* BEM Example */
.card { }
.card__image { }
.card__title { }
.card__button { }
.card--featured { border: 2px solid gold; }
.card__button--disabled { opacity: 0.5; }

Why BEM?

Self-Documenting

Class names explain structure

No Conflicts

Unique, scoped names

Reusable

Components work anywhere

Team Friendly

Everyone follows same pattern

CSS Limitations

Problems that made CSS hard to maintain at scale

Traditional CSS Problems

No Variables

Change brand color? Update 100+ places manually.

No Nesting

Repeat parent selectors over and over.

No Logic

No conditions, loops, or calculations.

Copy-paste same patterns everywhere.

/* Old CSS: Repetition everywhere */

.btn-secondary { background: #64748b; color: white; padding: 10px 20px; }

Modern CSS Solutions

CSS has evolved to address many limitations

CSS Variables

:root {
  --primary: #2563eb;
  --spacing: 1rem;
}
.btn { background: var(--primary); }

calc() Function

.sidebar {
  width: calc(100% - 250px);
  padding: calc(var(--spacing) * 2);
}

CSS Variables are now supported in all modern browsers and can even be changed with JavaScript!

Sass / SCSS

CSS with superpowers — a preprocessor that compiles to CSS

What is Sass?

Syntactically Awesome Style Sheets — extends CSS with programming features

You Write (.scss)

$primary: #2563eb;

.nav {
  background: $primary;
  a {
    color: white;
    &:hover { opacity: 0.8; }
  }
}

Compiles To (.css)

.nav { background: #2563eb; }
.nav a { color: white; }
.nav a:hover { opacity: 0.8; }

Key Sass Features

Variables

$font-size: 16px;
$primary: #2563eb;

Nesting

.card {
  .title { font-size: 1.5em; }
}

Mixins

@mixin flex-center {
  display: flex;
  align-items: center;
}

Partials

@import 'variables';
@import 'buttons';
