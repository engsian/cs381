Selecting Elements

// By ID (returns one element)
const header = document.getElementById('header');

// By class (returns collection)
const items = document.getElementsByClassName('item');

// Modern way - CSS selectors (recommended)
const header = document.querySelector('#header');      // First match
const items = document.querySelectorAll('.item');      // All matches
const btn = document.querySelector('button.primary');  // Complex selector

Best practice: Use querySelector / querySelectorAll — same syntax as CSS!

Modifying Elements

const el = document.querySelector('#box');

Adding Event Listeners

const btn = document.querySelector('#btn');

// Modern way (recommended)
btn.addEventListener('click', function(event) {
    console.log('Clicked!');
    console.log(event.target);  // The clicked element
});

    e.preventDefault();  // Stop default action (e.g., form submit)

});

// Remove listener

btn.addEventListener('click', handler);

Event Delegation

Handle many elements with one listener

// Instead of adding listener to each button...
const list = document.querySelector('#item-list');

list.addEventListener('click', (e) => {
    // Check what was actually clicked
    if (e.target.matches('.delete-btn')) {
        e.target.closest('.item').remove();
    }
    if (e.target.matches('.edit-btn')) {
        // Handle edit
    }
});

Why? Works for dynamically added elements. Better performance with many items.

Arrays

Array Methods (Powerful!)

const numbers = [1, 2, 3, 4, 5];

// map - transform each item
const doubled = numbers.map(n => n * 2);  // [2, 4, 6, 8, 10]

// filter - keep items that pass test
const evens = numbers.filter(n => n % 2 === 0);  // [2, 4]

// reduce - combine into single value
const sum = numbers.reduce((acc, n) => acc + n, 0);  // 15

// forEach - loop (no return)
numbers.forEach(n => console.log(n));

// Chaining
const result = numbers
    .filter(n => n > 2)
    .map(n => n * 10);  // [30, 40, 50]

// Two ways to create a regex
const regex1 = /pattern/flags;           // Literal (preferred)

Basic Regex Patterns

/abc/ — Exact match "abc"
/./ — Any single character
/\d/ — Any digit (0-9)
/\w/ — Word char (a-z, A-Z, 0-9, _)
/\s/ — Whitespace
/[abc]/ — a, b, or c
/[^abc]/ — NOT a, b, or c
/[a-z]/ — Range a to z
/^start/ — Starts with "start"
/end$/ — Ends with "end"

Tip: Use uppercase for negation: \D = non-digit, \W = non-word, \S = non-whitespace

Quantifiers

How many times a pattern should match

* — Zero or more
+ — One or more
? — Zero or one (optional)
{n} — Exactly n times
{n,} — n or more times
{n,m} — Between n and m times 
/a+/       // One or more 'a'  → matches "aaa"
/colou?r/  // 'u' is optional → matches "color" or "colour"
/\d{3}/    // Exactly 3 digits → matches "123"
/\d{2,4}/  // 2 to 4 digits   → matches "12", "123", "1234"

Regex Flags

g
Global
Find all matches
i
Case Insensitive
Ignore case
m
Multiline
^ and $ per line 
/hello/i.test('Hello');  // true (case insensitive)

'cat bat rat'.match(/at/g);  // ['at', 'at', 'at'] (all matches)

/hello/gi  // Combine flags: global + case insensitive

Regex Methods

const str = 'Hello World 123';
const regex = /\d+/;

// test() - Returns true/false
regex.test(str);                // true

// match() - Returns array of matches
str.match(/\d+/);               // ['123']
str.match(/\d/g);               // ['1', '2', '3']

// replace() - Replace matches
str.replace(/World/, 'JS');     // 'Hello JS 123'
str.replace(/o/g, '0');         // 'Hell0 W0rld 123'

// search() - Returns index of first match
str.search(/World/);            // 6

// split() - Split string by pattern
'a,b;c|d'.split(/[,;|]/);      // ['a', 'b', 'c', 'd']

Practical Examples

// Email validation
const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
emailRegex.test('user@example.com');  // true

// Phone number (Saudi format: 05xxxxxxxx)
const phoneRegex = /^05\d{8}$/;
phoneRegex.test('0512345678');  // true

// Password (min 8 chars, 1 uppercase, 1 lowercase, 1 number)
const passwordRegex = /^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).{8,}$/;
passwordRegex.test('Password1');  // true

// Extract hashtags from text
const text = 'Learning #JavaScript and #WebDev today!';
const hashtags = text.match(/#\w+/g);  // ['#JavaScript', '#WebDev']

// Remove extra whitespace
'  too   many   spaces  '.replace(/\s+/g, ' ').trim();  // 'too many spaces'

Groups and Capturing

// Capturing groups with ()
const dateRegex = /(\d{4})-(\d{2})-(\d{2})/;
const match = '2024-01-15'.match(dateRegex);
// match[0] = '2024-01-15' (full match)
// match[1] = '2024' (year)
// match[2] = '01' (month)
// match[3] = '15' (day)

// Named groups (?...)
const namedRegex = /(?\d{4})-(?\d{2})-(?\d{2})/;
const result = '2024-01-15'.match(namedRegex);
console.log(result.groups.year);   // '2024'
console.log(result.groups.month);  // '01'

// Replace with captured groups
'John Smith'.replace(/(\w+) (\w+)/, '$2, $1');  // 'Smith, John'

Use groups to extract specific parts of a match or rearrange text!

Destructuring & Spread

// Array destructuring
const [first, second, ...rest] = [1, 2, 3, 4, 5];
// first=1, second=2, rest=[3,4,5]

// Object destructuring
const { name, age, city = 'Unknown' } = user;

sum(1, 2, 3, 4);  // 10

Async JavaScript

Handling operations that take time

The Problem: Blocking Code

1 Request data from server
2 Wait 2 seconds... (page frozen!)
3 Display data

Solution: Async code runs in background. Page stays responsive. Code continues when data arrives.

Traditional Solution: Threads

Multi-Threading (Java, C++)

  • Spawn separate threads for tasks
  • Shared memory between threads
  • Complex: locks, mutexes, semaphores
  • Risk: race conditions, deadlocks
  • Hard to debug and maintain

JavaScript Event Loop

  • Single-threaded with event queue
  • Non-blocking I/O operations
  • No shared state problems
  • Callbacks, Promises, async/await
  • Easier to reason about

JavaScript's approach: Instead of managing threads, you write code that says "when this completes, do that" — the runtime handles the complexity!

Evolution of Async JavaScript

Callbacks
Nested, hard to read
Promises
Chainable, cleaner
async/await
Reads like sync!

Callback Hell

getUser(id, function(user) {
    getOrders(user, function(orders) {
        getDetails(orders[0], function(details) {
            // 3 levels deep!
        });
    });
});
  • Each async call nests inside previous
  • Forms "pyramid of doom"
  • Error handling is messy

async/await

const user = await getUser(id);
const orders = await getOrders(user);
const details = await getDetails(orders[0]);
// Flat and readable!
  • await pauses until complete
  • Code runs top-to-bottom
  • Use try/catch for errors

Promises

// A Promise represents a future value
const promise = new Promise((resolve, reject) => {
    // Async operation
    setTimeout(() => {
        resolve('Success!');  // or reject('Error!')
    }, 1000);
});

// Using a promise
promise
    .then(result => {
        console.log(result);  // 'Success!'
    })
    .catch(error => {
        console.error(error);
    })
    .finally(() => {
        console.log('Done');  // Always runs
    });

async/await (Modern Way)

// async function returns a Promise
async function fetchUser() {
    try {
        const response = await fetch('/api/user');
        const user = await response.json();
        return user;
    } catch (error) {
        console.error('Failed:', error);
    }
}

// Using it
const user = await fetchUser();
console.log(user.name);

// Or with .then()
fetchUser().then(user => console.log(user.name));

await pauses until Promise resolves. Code reads like synchronous but runs async!

Fetch API

// GET request
const response = await fetch('https://api.example.com/users');
const users = await response.json();

// POST request
const response = await fetch('https://api.example.com/users', {

JSON

The universal language of data exchange

What is JSON?

JavaScript Object Notation

What is it?

  • Lightweight text-based data format
  • Human-readable and writable
  • Language-independent
  • Based on JavaScript object syntax

Why use it?

  • Standard for web APIs
  • Easy to parse and generate
  • Supported by all languages
  • Smaller than XML

Key insight: JSON is how frontend and backend communicate. When you fetch data from an API, it's almost always JSON!

JSON Syntax Rules

{
    "name": "Ahmed",
    "age": 25,
    "isStudent": true,
    "courses": ["CS381", "CS201"],
    "address": {
        "city": "Yanbu",
        "country": "Saudi Arabia"
    },
    "graduated": null
}

Strict Rules

  • Keys must be in "double quotes"
  • Strings must use "double quotes"
  • No trailing commas allowed
  • No comments allowed
  • No undefined values
  • No functions or methods

JSON Data Types

String
"Hello"
Number
42, 3.14, -10
Boolean
true, false
null
null

Not allowed in JSON: undefined, functions, Date objects, NaN, Infinity, single quotes, comments

JSON.stringify()

Convert JavaScript object → JSON string

const user = { name: 'Ahmed', age: 25, active: true };

// Basic stringify
JSON.stringify(user);
// '{"name":"Ahmed","age":25,"active":true}'

// Pretty print with 2-space indentation
JSON.stringify(user, null, 2);
// {
//   "name": "Ahmed",
//   "age": 25,
//   "active": true
// }

// Select specific properties only
JSON.stringify(user, ['name', 'age']);
// '{"name":"Ahmed","age":25}'

Use cases: Sending data to server, storing in localStorage, debugging (pretty print)

JSON.parse()

Convert JSON string → JavaScript object

const jsonString = '{"name":"Ahmed","age":25,"active":true}';

// Basic parse
const user = JSON.parse(jsonString);
console.log(user.name);  // 'Ahmed'
console.log(user.age);   // 25

// IMPORTANT: Always handle parse errors!
try {
    const data = JSON.parse(invalidJson);
} catch (error) {
    console.error('Invalid JSON:', error.message);
}

// Common mistake - parsing already-parsed object
const obj = { name: 'Test' };
JSON.parse(obj);  // ERROR! Already an object, not a string

Working with Nested JSON

// API response with nested data
const response = {
    "status": "success",
    "data": {
        "user": {
            "id": 123,
            "name": "Ahmed",
            "courses": [
                { "code": "CS381", "grade": "A" },
                { "code": "CS201", "grade": "B+" }
            ]
        }
    }
};

// Accessing nested data
const userName = response.data.user.name;           // 'Ahmed'
const firstCourse = response.data.user.courses[0];  // { code: 'CS381', grade: 'A' }
const courseCode = response.data.user.courses[0].code;  // 'CS381'

// Safe access with optional chaining (?.)
const grade = response.data?.user?.courses?.[0]?.grade;  // 'A'
const missing = response.data?.user?.address?.city;      // undefined (not error)

JSON with Fetch API

// GET - Receiving JSON from API
const response = await fetch('https://api.example.com/users');
const users = await response.json();  // Parse JSON response
console.log(users[0].name);

// POST - Sending JSON to API
const newUser = { name: 'Ahmed', email: 'ahmed@example.com' };

const response = await fetch('https://api.example.com/users', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json'  // Tell server we're sending JSON
    },
    body: JSON.stringify(newUser)  // Convert object to JSON string
});

const created = await response.json();  // Parse JSON response
console.log('Created user ID:', created.id);

Remember: response.json() parses the response. JSON.stringify() prepares data to send.
