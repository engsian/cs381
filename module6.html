Introduction to Databases

Why we need databases

Why Use a Database?

Without Database

  • Data stored in files (txt, JSON)
  • Hard to search and filter
  • No relationships between data
  • Difficult to handle concurrent users
  • No data integrity protection

With Database

  • Structured data storage
  • Fast queries with SQL
  • Define relationships between tables
  • Handle thousands of concurrent users
  • Data validation and constraints

Real world: Imagine storing 1 million user records in a text file vs. a database. Which would be faster to search?

What is MySQL?

About MySQL

  • Open-source relational database
  • Developed by Oracle Corporation
  • Most popular database for web
  • Uses SQL (Structured Query Language)
  • Cross-platform (Windows, Linux, Mac)

Who Uses MySQL?

  • Facebook
  • Twitter / X
  • YouTube
  • WordPress (powers 40%+ of web)
  • Netflix

Fun fact: MySQL is named after co-founder Michael Widenius's daughter, My. MariaDB (a fork) is named after his other daughter!

Primary Keys & Data Types

Primary Key

  • Uniquely identifies each row
  • Cannot be NULL
  • Cannot have duplicates
  • Usually auto-incremented (id)

Common Data Types

  • INT - Integer numbers
  • VARCHAR(n) - Variable text
  • TEXT - Long text
  • DATE / DATETIME
  • DECIMAL(p,s) - Precise numbers
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    balance DECIMAL(10, 2) DEFAULT 0.00,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

SQL Fundamentals

Structured Query Language

What is SQL?

Structured Query Language — the standard language for databases

CREATE
READ
UPDATE
DELETE
SELECT - Read data
INSERT - Add new records
UPDATE - Modify records
DELETE - Remove records
CREATE TABLE - Make new table
DROP TABLE - Delete table

Note: SQL keywords are case-insensitive (SELECT = select), but UPPERCASE is the convention.

SELECT - Reading Data

-- Select all columns from a table
SELECT * FROM users;

-- Select specific columns
SELECT name, email FROM users;

-- Filter with WHERE clause
SELECT * FROM users WHERE id = 1;
SELECT * FROM users WHERE name = 'Ahmed';

-- Multiple conditions
SELECT * FROM users WHERE age >= 18 AND status = 'active';
SELECT * FROM users WHERE country = 'SA' OR country = 'AE';

-- Pattern matching with LIKE
SELECT * FROM users WHERE email LIKE '%@gmail.com';
SELECT * FROM users WHERE name LIKE 'A%';  -- Starts with A

-- Sorting results
SELECT * FROM users ORDER BY name ASC;      -- A to Z
SELECT * FROM users ORDER BY created_at DESC;  -- Newest first

-- Limit results
SELECT * FROM users LIMIT 10;              -- First 10
SELECT * FROM users LIMIT 10 OFFSET 20;    -- Skip 20, get 10

INSERT - Adding Data

-- Insert a single record
INSERT INTO users (name, email)
VALUES ('Ahmed Ali', 'ahmed@example.com');

-- Insert with all columns specified
INSERT INTO users (name, email, age, status)
VALUES ('Sara Khan', 'sara@example.com', 25, 'active');

-- Insert multiple records at once
INSERT INTO users (name, email) VALUES
    ('Omar Hassan', 'omar@example.com'),
    ('Fatima Ali', 'fatima@example.com'),
    ('Khalid Ahmed', 'khalid@example.com');

Note: You don't need to specify id if it's AUTO_INCREMENT — MySQL generates it automatically!

UPDATE - Modifying Data

-- Update a specific record
UPDATE users SET email = 'new@example.com' WHERE id = 1;

-- Update multiple columns
UPDATE users
SET name = 'Ahmed Mohammed', age = 26
WHERE id = 1;

-- Update multiple records
UPDATE users SET status = 'inactive' WHERE last_login < '2024-01-01';

-- Increment a value
UPDATE products SET stock = stock - 1 WHERE id = 5;

Warning: Always use WHERE clause! Without it, ALL rows will be updated!

DELETE - Removing Data

-- Delete a specific record
DELETE FROM users WHERE id = 1;

-- Delete with condition
DELETE FROM users WHERE status = 'inactive';

-- Delete old records
DELETE FROM logs WHERE created_at < '2023-01-01';

-- Delete ALL records (be careful!)
DELETE FROM users;  -- Removes all data, keeps table structure

-- Drop entire table (removes table completely)
DROP TABLE users;

Danger: DELETE without WHERE removes ALL data! Always double-check before running.

Connecting PHP to MySQL

PDO - PHP Data Objects

MySQLi vs PDO

MySQLi

  • MySQL only
  • Procedural & OOP styles
  • Good for MySQL-specific features

PDO (Recommended)

  • Works with 12+ databases
  • Consistent API
  • Named placeholders
  • Better error handling

We'll use PDO — it's more flexible and follows modern PHP best practices. If you ever switch to PostgreSQL or SQLite, your code works with minimal changes!

PDO Connection

<?php
// Database configuration
$host = 'localhost';
$dbname = 'cs381_db';
$username = 'root';
$password = '';  // Empty for Laragon default

// Create PDO connection
try {
    $pdo = new PDO(
        "mysql:host=$host;dbname=$dbname;charset=utf8mb4",
        $username,
        $password,
        [
            PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,  // Throw exceptions
            PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,  // Return arrays
            PDO::ATTR_EMULATE_PREPARES => false  // Real prepared statements
        ]
    );
    echo "Connected successfully!";
} catch (PDOException $e) {
    die("Connection failed: " . $e->getMessage());
}
?>

CRUD Operations with PDO

Create, Read, Update, Delete

CREATE - Insert Data

<?php
require_once 'db.php';

// Method 1: Direct query (NOT recommended for user input!)
$pdo->exec("INSERT INTO users (name, email) VALUES ('Test', 'test@example.com')");

// Method 2: Prepared statement (SAFE!)
$stmt = $pdo->prepare("INSERT INTO users (name, email) VALUES (:name, :email)");
$stmt->execute([
    ':name' => 'Ahmed Ali',
    ':email' => 'ahmed@example.com'
]);

// Get the ID of the inserted record
$newId = $pdo->lastInsertId();
echo "New user created with ID: $newId";

// Insert with form data
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $stmt = $pdo->prepare("INSERT INTO users (name, email) VALUES (:name, :email)");
    $stmt->execute([
        ':name' => $_POST['name'],
        ':email' => $_POST['email']
    ]);
}
?>

READ - Fetch Data

<?php
require_once 'db.php';

// Fetch all records
$stmt = $pdo->query("SELECT * FROM users");
$users = $stmt->fetchAll();

foreach ($users as $user) {
    echo $user['name'] . " - " . $user['email'] . "<br>";
}

// Fetch single record by ID
$id = 1;
$stmt = $pdo->prepare("SELECT * FROM users WHERE id = :id");
$stmt->execute([':id' => $id]);
$user = $stmt->fetch();  // Returns one row or false

if ($user) {
    echo "Found: " . $user['name'];
} else {
    echo "User not found";
}

// Fetch with conditions
$stmt = $pdo->prepare("SELECT * FROM users WHERE status = :status ORDER BY name");
$stmt->execute([':status' => 'active']);
$activeUsers = $stmt->fetchAll();

// Count records
$count = $pdo->query("SELECT COUNT(*) FROM users")->fetchColumn();
echo "Total users: $count";
?>

UPDATE - Modify Data

<?php
require_once 'db.php';

// Update a record
$stmt = $pdo->prepare("UPDATE users SET name = :name, email = :email WHERE id = :id");
$stmt->execute([
    ':name' => 'Ahmed Mohammed',
    ':email' => 'ahmed.new@example.com',
    ':id' => 1
]);

// Check if update was successful
if ($stmt->rowCount() > 0) {
    echo "User updated successfully!";
} else {
    echo "No changes made or user not found.";
}

// Update from form submission
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $stmt = $pdo->prepare("UPDATE users SET name = :name, email = :email WHERE id = :id");
    $stmt->execute([
        ':name' => $_POST['name'],
        ':email' => $_POST['email'],
        ':id' => $_POST['id']
    ]);
    header("Location: users.php?success=updated");
    exit();
}
?>

DELETE - Remove Data

<?php
require_once 'db.php';

// Delete a record
$stmt = $pdo->prepare("DELETE FROM users WHERE id = :id");
$stmt->execute([':id' => 5]);

if ($stmt->rowCount() > 0) {
    echo "User deleted successfully!";
} else {
    echo "User not found.";
}

// Delete with confirmation (typical workflow)
if (isset($_GET['delete']) && isset($_GET['id'])) {
    $id = (int) $_GET['id'];  // Cast to integer for safety

    $stmt = $pdo->prepare("DELETE FROM users WHERE id = :id");
    $stmt->execute([':id' => $id]);

    header("Location: users.php?success=deleted");
    exit();
}
?>

Best practice: Always confirm before deleting! Use a modal or confirmation page.

Prepared Statements

Security against SQL Injection

SQL Injection Attack

What if a malicious user enters this as their username?

// DANGEROUS - Never do this!
$username = $_POST['username'];  // User enters: ' OR '1'='1
$sql = "SELECT * FROM users WHERE username = '$username'";

// This becomes:
// SELECT * FROM users WHERE username = '' OR '1'='1'
// Returns ALL users! Attacker bypasses login!

SQL Injection is one of the most common and dangerous web vulnerabilities. It can expose, modify, or delete your entire database!

Prepared Statements = Safe

<?php
// SAFE - Always use prepared statements!
$username = $_POST['username'];  // Even if user enters malicious input

// The placeholder :username is treated as DATA, not SQL code
$stmt = $pdo->prepare("SELECT * FROM users WHERE username = :username");
$stmt->execute([':username' => $username]);

// The malicious input becomes a literal string search
// It searches for a user literally named: ' OR '1'='1
// Which doesn't exist - attack fails!
?>

Vulnerable

$sql = "... WHERE id = $id";

Protected

$stmt->execute([':id' => $id]);

How Prepared Statements Work

Step 1
Prepare: SQL template sent to database with placeholders SELECT * FROM users WHERE id = :id
Step 2
Compile: Database parses and compiles the query structure
Step 3
Execute: Values sent separately and treated as DATA only :id => "1 OR 1=1" (treated as literal string)

Key insight: The SQL structure is fixed before data is added. User input can never change the query logic!

create.php - Add User

<?php
require_once 'db.php';
$error = '';

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $name = trim($_POST['name']);
    $email = trim($_POST['email']);

    if (empty($name) || empty($email)) {
        $error = 'All fields are required';
    } elseif (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
        $error = 'Invalid email format';
    } else {
        $stmt = $pdo->prepare("INSERT INTO users (name, email) VALUES (:name, :email)");
        $stmt->execute([':name' => $name, ':email' => $email]);
        header("Location: index.php");
        exit();
    }
}
?>
<!DOCTYPE html>
<html>
<body>
    <h1>Add New User</h1>
    <?php if ($error): ?><p style="color:red"><?= $error ?></p><?php endif; ?>
    <form method="POST">
        <label>Name: <input type="text" name="name" required></label><br>
        <label>Email: <input type="email" name="email" required></label><br>
        <button type="submit">Create User</button>
    </form>
    <a href="index.php">Back to list</a>
</body>
</html>

edit.php - Update User

<?php
require_once 'db.php';
$error = '';

// Get user data
$id = $_GET['id'] ?? 0;
$stmt = $pdo->prepare("SELECT * FROM users WHERE id = :id");
$stmt->execute([':id' => $id]);
$user = $stmt->fetch();

if (!$user) { die("User not found"); }

// Handle form submission
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $name = trim($_POST['name']);
    $email = trim($_POST['email']);

    if (empty($name) || empty($email)) {
        $error = 'All fields are required';
    } else {
        $stmt = $pdo->prepare("UPDATE users SET name = :name, email = :email WHERE id = :id");
        $stmt->execute([':name' => $name, ':email' => $email, ':id' => $id]);
        header("Location: index.php");
        exit();
    }
}
?>
<h1>Edit User</h1>
<?php if ($error): ?><p style="color:red"><?= $error ?></p><?php endif; ?>
<form method="POST">
    <label>Name: <input type="text" name="name" value="<?= htmlspecialchars($user['name']) ?>" required></label><br>
    <label>Email: <input type="email" name="email" value="<?= htmlspecialchars($user['email']) ?>" required></label><br>
    <button type="submit">Update User</button>

delete.php - Remove User

<?php
require_once 'db.php';

$id = $_GET['id'] ?? 0;

if ($id) {
    // Verify user exists before deleting
    $stmt = $pdo->prepare("SELECT id FROM users WHERE id = :id");
    $stmt->execute([':id' => $id]);

    if ($stmt->fetch()) {
        // User exists, proceed with deletion
        $stmt = $pdo->prepare("DELETE FROM users WHERE id = :id");
        $stmt->execute([':id' => $id]);
    }
}

// Redirect back to list
header("Location: index.php");
exit();
?>

Enhancement: For production, use POST requests for delete operations and add CSRF token protection!
